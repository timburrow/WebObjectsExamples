This example application demonstrates the creation and use of WOFileUpload elements, allowing you to transport files from client machines to your application server (and the application, if required). This example will demonstrate many uploading methods, from the basic use (with both direct and component actions) to multiple file uploads on a single page, streaming or not.

There are also some additional pages demonstrating the use of the various streaming options for both raw uploading and raw downloading of data.

For demonstration purposes, several pages of the application are set to render images that are uploaded to the application. Attempting to display the uploaded files as images, requires reading them into memory. Thus, only the 'Component File to File Upload', 'Multiple Component File to File Upload in Form' and 'Direct Action Iterator Upload' examples may be used to upload arbitrarily large files. Note that the 'Direct Action Streaming Download' and 'Direct Action Request Content Streaming Upload' examples can be used with arbitrarily large files as well.

Since the images are not part of the demo, the uploads will be placed in a temporary folder () rather than in the project folder.
